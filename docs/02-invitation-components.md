# Toyga.kz — Компоненты страницы приглашения (`app/(guest)/invite/[slug]` или аналог)

> Публичная страница, которую открывает гость по ссылке. Mobile-first, тёмная тема `qara-ink` по умолчанию (у organizer'а в кабинете — светлая `paper`-тема из `01-core-components.md`). Это новые компоненты — в старом коде их не было в таком виде, `old/pages/invitation` использовать только как список требуемых данных (поля формы, структура события), не как визуальный референс.

## 1. `InvitationSealIntro` — экран открытия

Первый экран при заходе по ссылке, один раз за сессию (сохранять во `sessionStorage`/Zustand, не показывать повторно при скролле назад).

- Полноэкранный `qara-ink`, по центру — печать «Шаңырақ» (SVG, см. §4 в `00-brand-and-tokens.md`) в состоянии «закрыто»: линии не дорисованы.
- Тап/авто-через-1.2s → печать дорисовывается (SVG stroke-dashoffset анимация, 700ms) → раздвигается в стороны как створки → под ней проявляется `InvitationHero`.
- Внизу экрана — имена молодожёнов мелким eyebrow-текстом («приглашение для» / «сізге арналған шақыру») ещё до раскрытия печати, чтобы гость сразу понимал контекст.
- `prefers-reduced-motion` или повторный визит: экран пропускается, сразу `InvitationHero`.

## 2. `InvitationHero`

- Display-заголовок (Yeseva One, `text-display-xl`) — имена через декоративный разделитель (не «&», а маленький глиф печати), напр. «Айгерім ✦ Данияр».
- Под именами — тип события eyebrow-текстом (Той / Қыз ұзату / Сүндет той / Бесік той — берётся из `event.type`, тексты уже есть в модели БД `events`, 9 типов).
- Дата — `PT Mono`, крупно, вторым уровнем.
- Фоновая подложка: НЕ фото-стоки. Тонкая, малоконтрастная line-art версия орнамента қошқар-мүйіз по краям (opacity ≤ 0.08), либо загруженное фото пары с тёмным градиентным оверлеем `qara-ink` снизу для читаемости текста — оба варианта конфигурируются организатором в конструкторе шаблона.

## 3. `InvitationCountdown`

Не generic «дни:часы:минуты:секунды» блоки с рамками. Формат: крупные цифры `PT Mono` с золотым градиентом (единственное место, где разрешён `.gradient-text` из токенов), подписи снизу — eyebrow-текст на казахском/русском по локали. Между блоками — не двоеточие, а точка (типографски спокойнее для моно-шрифта).

## 4. `InvitationEventCard` (variant `seal` из Card)

Одна карточка = один смысловой блок: «Церемония», «Той», «Дресс-код», «Подарки». Иконка слева — тонкая золотая обводка (набор из lucide-react, обводка а не заливка через `strokeWidth={1.5}`). Заголовок — Golos Text 600, не Yeseva One (декоративный шрифт только в hero, здесь — читаемость важнее).

Карта/адрес — отдельная под-карточка с превью карты (статичное изображение или встроенная карта), кнопка «Построить маршрут» — `variant="wine"` из Button.

## 5. `RsvpForm`

Ключевой конверсионный компонент — максимальная скорость заполнения на телефоне:

- Шаг 1: «Придёте?» — три большие тач-таргет кнопки (не radio/select): Да / Нет / Пока не знаю — минимум `h-14`, полная ширина на мобильном, стиль как Badge-статусы из §Badge.
- Шаг 2 (только если «Да»): имя, количество гостей (степпер +/-, не text-input для числа), опционально пожелание/комментарий.
- Кнопка отправки — `variant="default"` (золотая), текст меняется по контексту: «Подтвердить приход» / «Отправить ответ».
- После отправки — состояние успеха: печать «Шаңырақ» в мини-версии проигрывает короткую анимацию подтверждения (переиспользуем SVG из intro, не новый чек-марк).
- Валидация инлайн (react-hook-form + zod, уже в стеке) — ошибка текстом `torgyn-wine` под полем, без красных рамок/иконок-крестиков.

## 6. `InvitationGallery`

Если у события есть медиа (`event_addresses`/media таблицы) — masonry-сетка 2 колонки на мобильном, лайтбокс при тапе. Рамка вокруг превью — 1px `saukele-gold/20`, при открытии лайтбокса фон затемняется до `qara-ink 90%`.

## 7. `GuestbookSection` (пожелания гостей)

Список сообщений — не чат-бабблы. Каждое пожелание — тонкая карточка `seal`-стиля без тени, разделитель между записями — глиф печати (§4 токенов) вместо `<hr>`. Имя автора — `PT Mono`, само сообщение — Golos Text.

## 8. Общие правила для секции

- Между всеми секциями компонент `SectionDivider` — единственный переиспользуемый разделитель: маленький центрированный SVG печати 24×24px, `opacity-40`. Никаких `border-t` линий на всю ширину — это уже стёрлось до generic-паттерна в старой версии.
- Vertical rhythm: `py-16` между секциями на мобильном, `py-24` от `md:` — достаточно воздуха, это люкс-бренд, не плотный лендинг.
- Каждая секция — `scroll-reveal` (см. Motion в токенах), но `InvitationCountdown` не анимируется при скролле (это единственный live-элемент, дёргать его дополнительной анимацией — визуальный шум).
