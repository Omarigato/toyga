# Toyga.kz — Кабинет организатора и общий каркас (`widgets/navbar`, `widgets/footer`, `app/(dashboard)/*`, `app/(auth)/*`)

> Тема кабинета — светлая `paper` (`ac-parqyt`), в отличие от тёмного публичного приглашения. Это осознанный контраст бренда: гость видит «бархат и золото ночью», организатор работает «за письменным столом при дневном свете» — тот же бренд, два режима одного материала (бумага/ткань).

## `Navbar` (`widgets/navbar`)

- Фон: `ac-parqyt/95` + `backdrop-blur(12px)` (переиспользуем `.glass`, обновлённый в токенах).
- Логотип — не текстовый «Toyga», а мини-версия печати «Шаңырақ» (16–20px) + wordmark Golos Text 600 рядом.
- Активный пункт меню — не подчёркивание, а точка-глиф печати под пунктом (2px) цвета `saukele-gold`.
- На публичных страницах (не в кабинете) навбар версия тёмная, прозрачная, становится `qara-ink/90` при скролле вниз > 80px.

## `Footer` (`widgets/footer`)

Тёмный всегда (`qara-ink`), независимо от темы страницы — footer это «подпись мастера» под любым документом. Печать «Шаңырақ» водяным знаком в правом нижнем углу, opacity 0.06, размер ~200px, частично обрезана краем экрана.

## Auth-формы (`app/(auth)/login`, `.../register`)

- Не полноэкранная форма по центру пустого экрана — слева/сверху (mobile: сверху) декоративная панель с печатью и цитатой/tagline на казахском, справа/снизу — форма на `paper`-карточке.
- Поля — как в `01-core-components.md`, `h-12`, крупные тач-таргеты (много пользователей заходят с телефона по ссылке из WhatsApp).
- OTP-инпут (используется в проекте, `auth/otp`) — 6 отдельных квадратных полей `PT Mono`, разделитель после 3-й цифры — точка, не тире.

## `DashboardOverviewCard` — карточки метрик на главной кабинета

- `variant="paper"`, крупное число `PT Mono` + eyebrow-лейбл сверху (не снизу — казахская типографика для чисел выигрывает, когда подпись предшествует).
- Тренд (+/-) — цвет `saukele-gold` для роста, `torgyn-wine` для падения — не зелёный/красный светофор (последовательность с Badge/RSVP статусами).

## `TemplateGalleryCard` (`app/(dashboard)/templates`)

- Превью шаблона — full-bleed изображение сверху карточки, скруглено только сверху (`rounded-t-2xl`), низ карточки — `paper` подложка с названием шаблона (Golos Text 600) и типом события (eyebrow).
- Premium-шаблоны — Badge «Premium» из §Badge поверх превью, левый верхний угол, с мини-иконкой печати.
- Hover (desktop): превью получает тонкую золотую обводку `1px saukele-gold/40`, лёгкий подъём `translateY(-2px)`, 150ms.
- Кнопка выбора — `variant="wine"`, появляется только при hover/фокусе на десктопе; на мобильном — всегда видна внизу карточки (нет hover-состояния на тач-устройствах, это частая ошибка в старой версии, которую нужно исправить).

## `EventListRow` (список событий пользователя, `app/(dashboard)/events`)

- Табличный/карточный гибрид: на мобильном — карточка `paper` в столбик, на `md:` — строка таблицы.
- Статус события (черновик / опубликовано / архив) — тот же паттерн Badge, но с нейтральной палитрой: черновик = `boz-dala` контур, опубликовано = `kok-tengri` заливка (единственное применение tengri-цвета как статус-заливки — «активно, живёт»), архив = `boz-dala/50` заливка приглушённая.

## Общие правила каркаса

- Кабинет никогда не использует Yeseva One для длинных текстов — только Golos Text. Декоративный шрифт зарезервирован за гостевой стороной продукта (см. `00-brand-and-tokens.md`, правило пары).
- Пустые состояния (нет событий, нет гостей) — печать «Шаңырақ» в разомкнутом виде (SVG вариант «до раскрытия» из intro-компонента) + текст-приглашение к действию, не generic иллюстрация из библиотеки undraw/humaaans.
- Загрузка/skeleton — см. `01-core-components.md` §Skeleton, единый паттерн по всему кабинету.
