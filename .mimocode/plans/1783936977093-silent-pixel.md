# Toyga.kz — Полный анализ проекта

## Что это за проект

**Toyga.kz** — казахстанская платформа цифровых свадебных/праздничных приглашений. Слово "тойға" означает "на свадьбу" на казахском языке. Пользователи выбирают шаблон, создают приглашение, отправляют гостям через WhatsApp и собирают RSVP.

**Домен в продакшене:** `https://toyga.kz`

---

### Архитектурные паттерны

1. **Clean Architecture (частичная)** — API → Services → Repositories → DB. Интерфейсы в `interfaces/`, реализации в `postgres/`
2. **Два стиля API** — современный (`withHandler()`) и legacy (ручные вызовы)
3. **Shared Validation** — Zod схемы в `src/core/validation/` используются и фронтендом, и API
4. **Manual DI** — `factories.ts` создаёт синглтоны репозиториев и сервисов
5. **Cookie-based Auth** — httpOnly JWT + Bearer header fallback
6. **Google Drive как CDN** — все медиа грузятся туда, отдаются через `/api/media/:fileId`

## API Endpoints (REST)

### Auth
- `POST /api/auth/register` — регистрация email+пароль
- `POST /api/auth/login` — вход email+пароль
- `POST /api/auth/logout` — выход
- `GET /api/auth/me` — текущий пользователь
- `POST /api/auth/otp-request` — запрос OTP через WhatsApp (rate-limit: 3/2min)
- `POST /api/auth/otp-verify` — верификация OTP

### Admin
- `POST /api/admin/login` — вход админа
- `GET/DELETE /api/admin/me` — информация/выход админа

### Templates & Categories
- `GET /api/templates` — список (публичный, фильтр по категории)
- `GET/POST /api/templates`, `GET/PUT/DELETE /api/templates/:id` — CRUD (admin)
- `GET /api/categories` — список (публичный)
- `POST/PUT /api/categories` — CRUD (admin)

### Events & Invitations
- `GET /api/invitations` — все (admin)
- `POST /api/invitations` — создание (user)
- `GET /api/invitations/:slug` — публичный доступ

### Guests / RSVP
- `GET/POST /api/guests/:invitationId` — список/отправка RSVP

### Orders
- `GET/POST /api/orders` — список/создание
- `PUT /api/orders?id=X` — обновление статуса

### Media
- `POST /api/upload` — загрузка в Google Drive
- `GET/DELETE /api/media/:fileId` — прокси/удаление

## База данных (17 таблиц)

### Аутентификация
- `users` — пользователи (multi-provider: password, phone_otp, google)
- `otp_codes` — OTP коды (хешированные, TTL 5мин, макс 3 попытки)
- `admins` — админы

### Каталог
- `categories` — категории шаблонов (slug, title_kk, sort_order)
- `templates` — шаблоны (title, price, preview_img, content JSONB, design_tokens JSONB)
- `template_assets` — медиа шаблонов (audio/video/image)

### Мероприятия
- `events` — события (9 типов: wedding, kyz_uzatu, sundet, tusaukeser, merey, besik, betashar, corporate, other)
- `event_addresses` — адреса (lat/lng)
- `event_invitations` — сохранённые дизайны (JSONB)
- `event_media` — медиа события

### Гости
- `guest_contacts` — список гостей с персональными slug
- `notification_jobs` — очередь рассылки WhatsApp
- `surveys` — RSVP ответы (yes/no/maybe)
- `survey_guests` — доп. гости

### Коммерция
- `orders` — заказы (pending/paid/cancelled/refunded)
- `payment_settings` — настройки оплаты (manual/Kaspi)
- `render_jobs` — очередь рендера видео

### Аудит
- `audit_logs` — журнал действий

**Особенности:** BIGSERIAL PK, soft deletes (`deleted_at`), auto-update триггеры (`set_updated_at()`)

## Доменная модель

### Бизнес-процесс

1. Админ создаёт категории + шаблоны
2. Пользователь регистрируется (email или WhatsApp OTP)
3. Создаёт мероприятие: выбор шаблона, ввод данных → получает slug `/i/{slug}`
4. Гость получает ссылку через WhatsApp
5. Гость видит: обложка + обратный отсчёт, детали, карта, аудио, RSVP-форма
6. Гость отправляет RSVP (да/нет/может + кол-во гостей + сообщение)
7. Админ видит все ответы в дашборде

### Типы мероприятий (казахская культура)
- **Wedding** (Үйлену той) — свадьба
- **Kyz Uzatu** (Қыз ұзату) — проводы дочери
- **Sundet** (Сүндет той) — обрезание
- **Tusaukeser** (Тұсаукесер) — первый шаг ребёнка
- **Merey** (Мерей той) — юбилей
- **Besik** (Бесік той) — колыбельная
- **Betashar** (Беташар) — открытие лица невесты
- **Corporate** — корпоратив

### Ценообразование
- Шаблон: `price` (KZT, 0 = бесплатно) + `extra_price` (обычно +400 KZT за кастомизацию)

## Дизайн-система

### Цвета (OKLCH)
- **background** — нейтральные (0-100% lightness, chroma 0)
- **primary** — зелёный (hue 145, wedding/festive feel)
- **accent** — золотистый/янтарный (hue 80)
- **secondary** — голубой/бирюзовый (hue 195)
- **foreground** — нейтральные для текста

### Шрифты
- **Display/Heading:** Cormorant Garamond (elegant serif) — fallback: Georgia
- **Body:** Inter (clean sans-serif) — fallback: system-ui

### Компоненты
- `SpotlightCard` — интерактивная карточка с radial-gradient hover
- `Navbar` — responsive с mobile hamburger, WhatsApp CTA, соцсети
- `Footer` — бренд, навигация, WhatsApp CTA
- `Countdown` — обратный отсчёт
- `RSVPForm` — форма RSVP (да/нет/может + гости + сообщение)
- `FileUploader` — загрузка файлов с прогрессом

### Без тестов
- Нет тестового фреймворка (vitest, jest, mocha)
- Нет ни одного тестового файла

- Нет GitHub Actions / GitLab CI / других пайплайнов

- Нет husky, lint-staged, commitlint

- `_auth.ts`, `_db.ts` — старые помощники, используемые рядом с現代енным `_core/`
