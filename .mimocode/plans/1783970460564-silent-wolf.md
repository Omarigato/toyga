# Toyga.kz Frontend Rebuild Plan

## Current State

- **Framework**: Next.js 16.2.10 (App Router) — already in place, no migration needed
- **Stack**: React 19, TailwindCSS v4, Zustand, TanStack Query, Framer Motion, React Hook Form + Zod
- **Problem**: All colors use generic amber/stone palette, fonts use Inter, no brand identity. The existing code is a typical AI-generated template, not the Toyga brand.

## What Changes

Replace the entire visual layer with the Toyga brand design system from `docs/00-brand-and-tokens.md`:
- **Colors**: amber/stone → qara-ink, ac-parqyt, saukele-gold, torgyn-wine, boz-dala, kok-tengri
- **Fonts**: Inter → Yeseva One (display), Golos Text (body), PT Mono (utility), Forum (eyebrow)
- **Components**: Add brand variants (paper/seal cards, wine button, seal SVG)
- **Pages**: Rebuild all pages with brand identity, add missing pages

## Implementation Plan

### Phase 1: Foundation (globals.css + fonts + tokens)

**File: `src/app/globals.css`**
- Replace all amber/stone CSS variables with brand tokens
- Add Google Fonts imports (Yeseva One, Golos Text, PT Mono, Forum)
- Define font family variables (`--font-display`, `--font-body`, `--font-mono`, `--font-eyebrow`)
- Add brand-specific utility classes: `.glass`, `.gradient-text`, `.scroll-reveal`, `.section-divider`
- Add `shadow-seal` custom shadow
- Remove all old amber/stone references

**File: `src/app/layout.tsx`**
- Add Google Fonts link tags in `<head>`
- Replace `bg-white text-stone-900` with brand tokens
- Add proper lang attribute and font class to `<html>`
